# 📖 Pagekin

> AI-powered personalized storybooks for children — complete with illustrations, read-aloud, voice recording, and PDF export.

**[Live Demo →](https://storytime-ai-eta.vercel.app)**

---

## What It Does

Pagekin lets parents and children create personalized AI-generated storybooks. Enter a child's name, age, and a theme — or hit **Surprise me!** to get an AI-suggested one — and Pagekin builds the story page by page, with a watercolor-style illustration for each scene.

- **Personalized stories** — the child's name, age range, and theme shape the vocabulary, sentence complexity, and narrative
- **AI watercolor illustrations** — each page gets a unique image generated by HuggingFace FLUX.1-schnell
- **Read aloud** — Gemini TTS reads each page in a natural voice, or play the full story hands-free
- **Voice recording** — parents can record their own narration per page; recordings replace AI voice during playback
- **PDF export** — download the finished book as an A5 PDF with a cover page, illustrations, and page numbers
- **Saved sessions** — up to 5 stories are saved locally so you can pick up where you left off

## Technical Highlights

**Server-side API proxy** — All AI keys live in Vercel environment variables and never reach the browser. A three-key rotation system (`_geminiWithRotation.js`, `_hfWithRotation.js`) cycles through keys on 429s, with silent fallback so quota errors degrade gracefully rather than crashing.

**Per-page on-demand generation with background prefetch** — Pages are generated one at a time as the reader advances. While the user reads page N, the app silently prefetches text and kicks off image generation for pages N+1 and N+2, so navigation feels instant.

**Stale-closure-safe async loops** — `storyPagesRef` and `currentPageIndexRef` are kept in sync with state via `useEffect`. The play-all and record-all loops read from refs so they always see the latest page data without stale closure bugs.

**Memory-safe voice recording** — Recordings use `URL.createObjectURL` for in-memory playback and call `URL.revokeObjectURL` on deletion or replacement to prevent memory leaks. Blob URLs are intentionally excluded from `localStorage` persistence since they don't survive page reload.

**Web Audio API playback** — Gemini TTS returns base64-encoded PCM audio. The app decodes it into an `AudioBuffer` and plays it through the Web Audio API, giving precise control over start, stop, and `AudioContext` lifecycle (including handling browser autoplay suspension).

## Getting Started

**Prerequisites:** Node.js 18+ and a free [Google Gemini API key](https://aistudio.google.com/app/apikey)

```bash
# 1. Clone the repo
git clone https://github.com/ManchDi/pagekin.git
cd pagekin

# 2. Install dependencies
npm install

# 3. Add your API key
echo "GEMINI_API_KEY=your_key_here" > .env.local

# 4. Run locally
npm run dev
```

> **Note:** Image generation requires a HuggingFace API key (`HF_API_KEY` in `.env.local`). You can disable illustrations in the app's UI to skip this.

## Project Structure

```

│   ├── _geminiWithRotation.js  # Gemini key rotation helper
│   ├── _hfWithRotation.js      # HuggingFace key rotation helper

├── services/
│   ├── geminiService.ts        # API proxy calls + QuotaError class
│   └── pdfService.ts           # jsPDF A5 export
├── App.tsx                     # All state, story logic, session management
└── types.ts                    # StoryConfig, StoryPage, SavedSession types
```
